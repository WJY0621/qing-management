# 添加新食材计划

为了满足您添加更多食材（水果、蔬菜、肉类、调料）的需求，我将执行以下操作：

## 1. 更新前端静态数据 (`constants.tsx`)
目前 `RecipeEditor.tsx` (食谱编辑器) 直接使用了本地的 `MOCK_INGREDIENTS` 常量列表。为了让您能在 APP 中立即看到并使用这些新食材，我将把它们添加到 `src/constants.tsx` 文件中。

**计划添加的类别：**
*   **水果**: 苹果、葡萄、橙子、草莓、西瓜、桃子、梨等。
*   **蔬菜**: 白菜、生菜、胡萝卜、洋葱、青椒、茄子、冬瓜等。
*   **肉类/水产**: 猪肉(瘦)、鸭肉、虾仁、鳕鱼、鲈鱼等。
*   **油脂/调料**: 橄榄油、花生油、大豆油、猪油、黄油、酱油、蚝油、料酒等。

## 2. 生成数据库迁移脚本 (`supabase/add_ingredients.sql`)
虽然前端使用了静态数据，但为了保持后端数据库 (`public.ingredients` 表) 的完整性（特别是如果您将来要切换到服务端搜索），我将创建一个新的 SQL 脚本。

*   **文件**: `supabase/add_ingredients.sql`
*   **内容**: 包含一系列 `INSERT INTO public.ingredients ...` 语句。
*   **用途**: 您需要在 Supabase 的 SQL Editor 中运行此脚本，将这些新食材同步到云端数据库。

## 3. 验证
*   修改完成后，您进入“食谱” -> “新建食谱” -> “添加食材”时，就能搜索到这些新添加的食材了。
